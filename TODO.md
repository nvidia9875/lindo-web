## T2. Safari で Works が表示されない ✅ 完了

**目的** iOS Safari で Works セクションが真っ白のまま出ず、画面に触れると出てくる問題を解消。

**原因（特定済み・再現確認済み）**
Safari のエンジン差ではなく、**「背の高い要素 × IntersectionObserver の threshold」** の問題。

- `template-parts/section-artists.php:21` — `<div class="sec-body rv d1">` が h2・リード文・カードグリッド**全体**を包んでいる。SP1カラムで **高さ 4949px**。
- `assets/js/main.js:37` — `{ threshold: 0.14, rootMargin: "0px 0px -8% 0px" }`
- `assets/css/lindo.css:1538-1543` — `.rv { opacity: 0 }`、`.rv.in` で初めて表示。

要素がビューポートより高いと、交差比率は `実効ルート高 ÷ 要素高` を超えられない。`-8%` を引くと実効ルートは `vh × 0.92` なので、発火条件は **`vh × 0.92 ≥ 0.14 × 4949` → `vh ≥ 753px`**。

| 端末状態 | innerHeight | 発火 |
|---|---|---|
| iPhone・URLバー展開時 | 約 660〜735px | **しない**（＝真っ白） |
| iPhone・URLバー収納後 | 約 844〜852px | する |
| PC Chrome 1440×900（3カラム・1752px） | 900 | する |

触ると出るのは、スクロールで URL バーが畳まれて `innerHeight` が跳ね上がり、比率が 0.14 を超えるため。PCは3カラムで要素が低く比率0.473なので再現しない＝**見逃されていた**。

**修正方針** threshold が背の高いコンテナに対して不適切。いずれか。

1. `threshold: 0` にする（1pxでも触れたら発火）… 最小変更・全 `.rv` に効く
2. `.rv` を背の高いラッパーではなく**中の要素**に付け替える
3. sentinel 要素を観測する

**1 が最小で確実**。ただし「少し見えたら即発火」になるので、リビールの見え方が変わる。`rootMargin` の下マージンを調整して発火位置を作る方が自然かもしれない（例 `threshold: 0` ＋ `rootMargin: "0px 0px -10% 0px"`）。

**同じ地雷が他にもある** `.rv` は全セクションのラッパーに付いており（About/Service/Contact 等）、**Works が一番高いので最初に露呈しただけ**。修正は `main.js` 側で一括で効かせる。

**完了条件** 375×660（URLバー展開相当の低いビューポート）で Works が触らずに表示される。他セクションのリビールも壊れていない。実機Safariでも確認。

## T3. 画像が途中で切れて表示される

**目的** 画像の切れを解消。**原因が2つあり、別物**なので分けて対応する。

### T3-A. ライトボックスで画像の下が切れる（PC・実バグ）✅ 完了

拡大表示は本来「全体を見せる」ものなのに、**PCで画像の下が切れている**。

- `assets/css/lindo.css:1214` — `.lb-img { max-height: 100% }` が **効いていない（no-op）**。
- 理由: `.lb-figure`（`lindo.css:1204-1211`）が `display:grid; place-items:center` で**行が auto サイズ**＝高さが不定。不定の親に対する `%` 指定は `none` 扱いになる。
- 結果、画像が原寸で描画され、`.lightbox { overflow:hidden }`（`lindo.css:1174`）ではみ出しが切り落とされる。

| ビューポート | 実測 |
|---|---|
| 1440×900 | 863×1280 の画像が原寸描画 → **下が 430px 切れる** |
| 1280×800 | → **下が 525px 切れる** |
| 390×844（SP） | `max-width:100%` が効くので切れない |

**＝ノートPC等（横幅は足りるが縦が1380px未満）で切れる。** SPで見ていたので気づかなかった。T2がSP限定なのと真逆。

**修正方針** 高さの制約を解決可能にする。どちらでも直ることを実証済み。
1. `.lb-figure { grid-template-rows: 100% }`（行を確定させる）
2. `.lb-img { max-height: ... }` を `dvh` ベースにする（`padding` 分を差し引く必要あり）

**完了条件** 1440×900 / 1280×800 / 390×844 で縦長画像が切れずに全体表示される。
